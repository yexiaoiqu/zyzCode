

本文专为 Git 零基础、GitHub 新手编写，已跳过安装步骤（默认你已成功安装 Git 并注册 GitHub 账号），重点覆盖 **Git 初始化→身份配置→本地仓库→GitHub 远程关联→核心操作** 全流程，命令可直接复制，步骤无坑，小白可直接跟着操作。

## 一、前置说明（必看）

1.  确认 Git 已安装：打开终端（Windows 打开 Git Bash、macOS/Linux 打开自带终端），输入 `git --version`，出现版本号（如 git version 2.45.1）即正常。

2.  确认 GitHub 账号已注册：已拥有 GitHub 账号（未注册可先访问 [https://github.com](https://github.com) 完成注册，再继续）。

3.  核心逻辑：Git 管理本地代码版本，GitHub 作为云端仓库备份/协作，全程跟着命令走，不用理解复杂原理。

## 二、第一步：配置 Git 身份（必须！）

首次使用 Git 必须配置用户名和邮箱，这会显示在你的所有代码提交记录中，建议与 GitHub 账号的用户名、邮箱保持一致（避免混乱）。

打开终端/Git Bash，输入以下两条命令（复制粘贴即可，替换引号内内容）：

```bash
# 全局配置（所有本地仓库都生效，推荐）
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

# 验证配置是否成功（可选，输入后可查看配置信息）
git config --list
```

示例：若你的 GitHub 用户名是 test123，邮箱是 test123@163.com，则命令为：

```bash
git config --global user.name "test123"
git config --global user.email "test123@163.com"
```

## 三、第二步：初始化本地 Git 仓库（核心第一步）

本地仓库就是你存放代码的文件夹，通过 Git 初始化后，才能记录文件的修改历史。步骤如下：

```bash
# 1. 新建一个文件夹（作为你的项目仓库，名字可自定义，如 my-first-project）
mkdir my-first-project

# 2. 进入这个文件夹（切换到仓库目录）
cd my-first-project

# 3. 初始化 Git 仓库（关键命令！执行后会生成一个隐藏的 .git 目录，管理版本记录）
git init
```

执行完 `git init` 后，终端会提示 “Initialized empty Git repository in ...”，说明本地仓库初始化成功。

注意：初始化后，这个文件夹里的所有文件（代码、文档等）都会被 Git 监控，后续修改都会被记录。

## 四、第三步：GitHub 新建远程仓库（用来备份本地代码）

本地仓库初始化后，需要在 GitHub 上新建一个远程仓库，用来备份本地代码、方便后续推送/拉取。操作步骤（纯鼠标点击，无需命令）：

1. 登录 GitHub（https://github.com），点击右上角的 **+** 图标，选择 **New repository**（新建仓库）。

2. 填写仓库信息（小白默认配置即可）：

3. Repository name（仓库名）：建议和本地仓库名一致（如 my-first-project），方便对应。

4. Description（描述，可选）：填写项目说明，如 “我的第一个 Git+GitHub 测试项目”。

5. 勾选 **Add a README file**（新手必选！避免后续推送报错）。

6. 其他选项默认（仓库类型选 Public，免费可用）。

7. 点击最下方的 **Create repository**，远程仓库就创建完成了。

8. 复制远程仓库地址（关键！后续关联本地需要）：新建完成后，页面会显示仓库地址，选择 **HTTPS** 协议，点击地址右侧的复制图标（📋），复制完整地址（格式：https://github.com/你的GitHub用户名/仓库名.git）。

## 五、第四步：本地仓库关联 GitHub 远程仓库

关联后，本地的代码才能推送到 GitHub 云端，执行以下命令（复制粘贴，替换地址为你刚才复制的远程仓库地址）：

```bash
# 关联远程仓库（origin 是远程仓库的默认别名，固定写法，不用修改）
git remote add origin https://github.com/你的GitHub用户名/你的仓库名.git

# 查看关联是否成功（可选，执行后会显示远程仓库地址）
git remote -v
```

成功提示：执行 `git remote -v` 后，会显示 origin 对应的两个地址（fetch 和 push），说明关联成功。

常见错误：若提示 “fatal: remote origin already exists”，说明已关联过，可输入 `git remote rm origin` 删除原有关联，再重新执行关联命令。

## 六、第五步：核心操作（提交本地代码→推送到 GitHub）

这是小白最常用的流程，本地修改代码后，同步到 GitHub 云端，步骤固定，记熟即可：

```bash
# 1. 查看文件状态（必做！查看本地有哪些文件被修改/新增）
git status

# 2. 将所有修改/新增的文件添加到暂存区（暂存区是提交的过渡区）
git add .  # 注意：add 后面有一个空格 + 点，代表“添加当前目录下所有文件”

# 3. 将暂存区的文件提交到本地仓库（-m 后面是提交说明，必填！要写清楚修改内容）
git commit -m "提交说明"  # 示例：git commit -m "初始化项目，添加README文件"

# 4. 首次推送：将本地仓库的代码推送到 GitHub 远程仓库
git push -u origin main  # 首次推送必须加 -u，绑定本地main分支和远程main分支

# 后续推送（第二次及以后，直接执行，无需加 -u）
git push
```

推送说明：首次推送时，会弹出 GitHub 登录窗口（或输入账号密码），按提示登录即可，登录成功后，代码会自动同步到 GitHub 远程仓库。

验证：推送完成后，刷新 GitHub 远程仓库页面，就能看到本地提交的文件，说明推送成功。

## 七、补充场景：已有 GitHub 项目（克隆到本地）

若你想把 GitHub 上已有的项目（别人的或自己的）下载到本地修改，无需初始化本地仓库，直接克隆即可：

```bash
# 1. 复制 GitHub 上目标仓库的 HTTPS 地址（步骤同第四步第4点）
# 2. 克隆仓库到本地（自动创建和仓库名一致的文件夹，无需手动新建）
git clone https://github.com/用户名/仓库名.git

# 3. 进入克隆后的项目文件夹（后续修改代码就在这个文件夹里）
cd 仓库名
```

## 八、小白常用核心命令速查（必背）

- 初始化本地仓库：`git init`

- 配置身份：`git config --global user.name "用户名"` / `git config --global user.email "邮箱"`

- 关联远程仓库：`git remote add origin 远程地址`

- 查看文件状态：`git status`

- 添加文件到暂存区：`git add .`

- 提交到本地仓库：`git commit -m "提交说明"`

- 首次推送：`git push -u origin main`

- 后续推送：`git push`

- 克隆远程仓库：`git clone 远程地址`

## 九、小白常见问题（避坑指南）

1. 推送时报错 “fatal: remote error”：原因是 GitHub 远程仓库有本地没有的内容（如勾选的 README 文件），解决方法：先拉取远程内容，再推送。
        `git pull origin main  # 拉取远程最新内容
    git push  # 再推送本地内容`

2. 提交说明写错了：修改最后一次提交的说明，执行以下命令：
`git commit --amend -m "正确的提交说明"`

3. 忘记配置身份，提交时报错：重新执行身份配置命令，再重新提交即可。

## 十、总结

小白入门核心流程：**配置身份 → 初始化本地仓库 → GitHub 新建远程仓库 → 关联远程 → add → commit → push**，掌握这一套流程，就能完成个人代码的本地管理和云端备份，满足日常开发需求。

后续可进阶学习：分支管理（多人协作必备）、SSH 免密登录（不用每次推送输密码）、代码回滚等。

## 十一、VS Code 中 Git 使用方法（小白友好，无需命令）

默认你已安装 VS Code，且已完成上述 Git 身份配置、本地仓库初始化/关联远程操作，VS Code 内置 Git 功能，全程鼠标操作，比终端命令更直观，适合小白。

### 1. 打开本地 Git 仓库（第一步）

1. 打开 VS Code，点击左侧菜单栏「文件」→「打开文件夹」，选择你已初始化的本地仓库文件夹（如 my-first-project）。

2. 打开后，点击左侧最下方的「源代码管理」图标（类似分支的图标，快捷键 Ctrl+Shift+G / Cmd+Shift+G），即可看到 Git 相关操作界面。

### 2. 核心操作：修改文件→暂存→提交→推送（对应终端命令）

全程无需输入命令，点击按钮即可完成，与前文终端流程完全对应，步骤如下：

1. 修改/新增文件：在 VS Code 中直接编辑仓库内的文件（如新建 test.txt、修改 README.md），编辑后保存（Ctrl+S / Cmd+S）。

2. 暂存文件（对应 `git add .`）：
        

源代码管理界面中，会显示“更改”列表（所有修改/新增的文件），点击列表上方的「+」号（暂存更改），即可将所有文件添加到暂存区；若只想暂存单个文件，点击该文件右侧的「+」号即可。
      

3. 提交文件（对应 `git commit -m "提交说明"`）：
        

暂存完成后，在界面上方的“消息”输入框中，填写提交说明（如“新增test.txt文件”），填写完成后，点击输入框右侧的「√」（提交），即可将暂存区文件提交到本地仓库。
      

4. 推送文件（对应 `git push`）：
        

提交完成后，点击源代码管理界面右上角的「↑」（推送）图标，首次推送会提示登录 GitHub（按提示登录即可），登录成功后，自动将本地代码推送到 GitHub 远程仓库，推送完成后会有成功提示。
      

### 3. 补充操作（常用）

- 拉取远程代码（对应 `git pull origin main`）：点击源代码管理界面右上角的「↓」（拉取）图标，即可拉取 GitHub 远程仓库的最新内容，避免协作时冲突。

- 克隆远程仓库（对应 `git clone`）：打开 VS Code，点击「克隆存储库」，粘贴 GitHub 仓库的 HTTPS 地址，选择本地保存路径，点击「克隆」，即可自动将远程仓库下载到本地。

- 查看提交历史：在源代码管理界面，点击「更多操作」（三个点图标）→「查看提交记录」，即可看到所有提交记录，包括提交人、时间、提交说明。

注意：1. 若 VS Code 提示“未检测到 Git”，重启 VS Code 即可（确保已安装 Git）；2. 首次使用 VS Code 推送时，登录 GitHub 后会自动关联本地与远程仓库，无需额外操作。

VS Code Git 操作核心逻辑：和终端流程一致，只是将命令替换为鼠标点击，更适合不熟悉终端的小白，熟练后可根据习惯选择终端或 VS Code 操作。
